# Rechera ナレッジパッケージ

このパッケージは、Rechera（リシェラ）女性向けSNS運用スクールのWebサイトを構築するための完全なナレッジベースとマスタープロンプトを提供します。

---

## 📦 パッケージ内容

このナレッジパッケージには、以下のドキュメントが含まれています。

### 1. **MASTER_PROMPT_RECHERA.md** ⭐ 最重要
AI開発者に投げるためのマスタープロンプト。このプロンプトを Claude や ChatGPT に投げることで、Recheraの MVP を構築できます。

**使用方法:**
1. このファイルの内容を全てコピー
2. AI開発ツール（Claude、ChatGPT、Cursor等）に貼り付け
3. 「このプロンプトに従ってRecheraのMVPを構築してください」と指示

### 2. **TECHNICAL_SPECIFICATION_RECHERA.md**
技術仕様書。コンポーネント設計、API仕様、データベーススキーマ、デプロイメント設定の詳細を提供します。

**使用タイミング:**
- 実装の詳細を確認したいとき
- コンポーネントの設計を理解したいとき
- API エンドポイントの仕様を確認したいとき

### 3. **IMPLEMENTATION_CHECKLIST_AND_DEPLOY_GUIDE.md**
実装チェックリストとCoolifyデプロイガイド。段階的に実装を進めるためのチェックリストと、Coolifyへのデプロイ手順を提供します。

**使用タイミング:**
- 実装の進捗を管理したいとき
- Coolifyへのデプロイを実行するとき
- トラブルシューティングが必要なとき

### 4. **rechera_integrated_knowledge_base.md**
統合ナレッジベース。デザインシステム、カラーパレット、タイポグラフィ、背景エフェクト、複数ページ構成、CMS統合の詳細を提供します。

**使用タイミング:**
- デザインの詳細を確認したいとき
- カラーパレットやフォントの仕様を確認したいとき

### 5. **rechera_background_effects_guide.md**
背景エフェクト実装ガイド。グラデーション背景、Blob アニメーション、パーティクルエフェクトの実装方法を提供します。

**使用タイミング:**
- 背景エフェクトの実装方法を確認したいとき
- アニメーションのコード例が必要なとき

### 6. **client_assets_analysis.md**
クライアント提供画像の分析。カラーパレット、ライフスタイル画像の詳細分析を提供します。

**使用タイミング:**
- クライアント提供画像の使用方法を確認したいとき
- カラーコードを確認したいとき

### 7. **クライアント提供画像（18枚）**
`/home/ubuntu/upload/` ディレクトリに保存されています。

## 🚀 クイックスタートガイド

### 最速で MVP を構築する方法

#### Step 1: マスタープロンプトを準備

```bash
# マスタープロンプトをコピー
cat MASTER_PROMPT_RECHERA.md | pbcopy  # Mac
cat MASTER_PROMPT_RECHERA.md | xclip -selection clipboard  # Linux
```

#### Step 2: AI開発ツールに貼り付け

1. Claude、ChatGPT、Cursor等のAI開発ツールを開く
2. マスタープロンプトを貼り付け
3. 以下の指示を追加

```
このプロンプトに従って、Recheraの MVP を構築してください。
実装チェックリストに従って、段階的に進めてください。
```

#### Step 3: クライアント提供画像をアップロード

AI開発ツールに、`/home/ubuntu/upload/` ディレクトリの画像をアップロードしてください。

#### Step 4: 実装の開始

AIが自動的に以下の作業を実行します：

1. Next.js プロジェクトの初期化
2. Strapi プロジェクトの初期化
3. デザインシステムの構築
4. コンポーネントの実装
5. ページの実装
6. API統合
7. レスポンシブデザインの実装
8. パフォーマンス最適化

#### Step 5: デプロイ

`IMPLEMENTATION_CHECKLIST_AND_DEPLOY_GUIDE.md` の「Coolify デプロイガイド」に従って、デプロイを実行してください。

## ⚠️ 重要な注意事項

### 1. 環境変数の管理

**絶対にハードコードしない:** すべての機密情報（データベースパスワード、JWT シークレット、AWS キー等）は環境変数で管理してください。

**Git にコミットしない:** `.env` ファイルを `.gitignore` に追加してください。

### 2. 画像の最適化

**WebP 変換必須:** すべての JPEG/PNG を WebP に変換してください。

**レスポンシブ画像:** 複数サイズ（320px, 640px, 1024px, 1920px）を生成してください。

### 3. パフォーマンス目標

- **Lighthouse Performance:** 90以上
- **ページロード速度:** 3秒以下
- **First Contentful Paint (FCP):** 1.8秒以下

### 4. アクセシビリティ

- **WCAG 2.1 AA準拠:** すべてのページがアクセシビリティ基準を満たすこと
- **キーボードナビゲーション:** すべての機能がキーボードで操作可能であること
- **スクリーンリーダー対応:** 適切な ARIA ラベルを使用すること

## 🐛 トラブルシューティング

### よくある問題と解決策

#### 問題: Docker イメージのビルドに失敗する

**解決策:**
1. ビルドログを確認
2. ローカル環境で `docker build` を実行し、エラーを特定
3. Dockerfile を修正してプッシュ

#### 問題: データベースに接続できない

**解決策:**
1. 環境変数が正しく設定されているか確認
2. Docker Compose ログで PostgreSQL の起動を確認
3. `docker-compose logs postgres` でエラーを確認

#### 問題: SSL証明書が発行されない

**解決策:**
1. DNS レコードを確認（`nslookup rechera.jp`）
2. ドメインが Coolify サーバーの IP アドレスを指しているか確認
3. DNS の伝播を待つ（最大24時間）
